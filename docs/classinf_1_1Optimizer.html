#include "inf/optimization/optimizer.h"

Class description

This class defines the interface for inf::BruteForceOpt and inf::TreeOpt.

It describes how to efficiently minimize an inf::DualVector, representing some \(\quovec = \{\quovec_\constraintname\}_{\constraintname\in\constraintlist}\in\totquovecspace\), on the events of some inf::Inflation, i.e., solving the minimization problem

\[ \min_{\infevent\in\infevents} \inner{\quovec}{\totconstraintmap(\detdistr\infevent)}_{\constraintlist}, \]

where \(\constraintlist \subset \infconstraints\) describes the constraints of an inflation problem (see inf::ConstraintSet).

◆ SearchMode

enum class inf::Optimizer::SearchMode
strong

Describes the search strategy: how should one tackle the problem of enumerating all relevant inflation events?

This is is used in in::Optimzier::get_optimizer() to conveniently select the subclass of inf::Optimizer that should be instantiated.

Enumerator
brute_force 

Simply loop over all events. Suitable for small inflations. See inf::BruteForceOpt.

tree_search 

The symmetrized events are stored as a tree and a tree search is performed to solve the minimization. See inf::TreeOpt.

◆ m_stop_mode

inf::Optimizer::StopMode inf::Optimizer::m_stop_mode
protected

See inf::Optimizer::StopMode. This attribute is set by inf::Optimizer::optimize(), so that it can be accessed within inf::Optimizer::get_pre_solution() overrides.

This is a member variable to be easily accessed by the child classes of inf::Optimizer. Otherwise, if it was just a parameter of inf::Optimizer::get_pre_solution(), it would also need to be forwarded to all the other child functions.
