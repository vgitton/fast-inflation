Module description

Support for gdb and soft/hard assertions.

Assertions

The hard assertions look like HARD_ASSERT_TRUE(X), HARD_ASSERT_EQUAL(X,Y), etc. Such assertions are always tripped, and throw an error indicating the file, line, and expected statement.

The soft assertions look like ASSERT_TRUE(X), ASSERT_EQUAL(X,Y), etc. Such assertions are only tripped in debug mode, which is internally controlled by the NDEBUG macro only defined in released mode. For instance:

#ifndef NDEBUG
// Here code only executed in debug mode
#else
// Here code only executed in release mode
#endif

However, the point of the debugging macros is to not have to use such #ifndef NDEBUG statements in applications. For those cases where this is actually convenient, one can use

DEBUG_RUN(
// Here code only executed in debug mode
int x = 0;
++x;
util::logger << x << util::cr;
)

Support for gdb

Gdb support is currently turned off (to speed up compilation, mostly). It can be brought back in the makefile by uncommenting the -g flag in the DEBUGFLAGS variable.

◆ BREAK_HERE

#define BREAK_HERE   util::break_here();

Use to set a breakpoint for gdb.

In ~/.gdbinit, place

set breakpoint pending on
break break_here

to be able to use the BREAK_HERE macro, which evaluates to a call to the dummy function void break_here() {} in debug mode, and to nothing in release mode. Then, gdb will break on the line where this macro was called.
