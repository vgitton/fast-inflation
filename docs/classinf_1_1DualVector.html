#include "inf/constraints/dual_vector.h"

Class description

Describes a dual vector \(\quovec\in\quovecspace{\infmarg}\) through its embedding \(\altquovecembed\infmarg(\quovec) \in \vecset\infmarg\).

Given an inflation constraint \(\constraintname = (\infmarg_0,\dots,\infmarg_{k-1},\infmargg) \in \infconstraints\), the group of constraint symmetries \(\marggroup\) denotes all the inflation symmetries of \(\distrgroup\) (see inf::Inflation::get_inflation_symmetries()) that leave both \(\infmarg_0 \cup \dots \cup \infmarg_{k-1}\) and \(\infmargg\) invariant. Then, for an inflation marginal \(\infmarg \in \{\infmarg_0\cup\cdots\infmarg_{k-1}\cup\infmargg, \infmargg\}\), the reduced set of marginal events is \(\repr{\events\infmarg \setminus \marggroup}\), and the quovec vector space \(\quovecspace\infmarg\) is the vector space of all functions \(\repr{\events\infmarg\setminus\marggroup} \to \R\).

Orbits

The inf::DualVector first task is to compute the orbits \(\events\infmarg \setminus \marggroup\). This then allow to go from the inf::Quovec representation \(\quovec \in \quovecspace\infmarg\) to the inf::EventTensor representation \(\altquovecembed\infmarg(\quovec)\in\vecset{\infmarg}\). Most of the relevant code is inherited from the inf::Orbitable and inf::TensorWithOrbits classes to avoid duplication of code that is also used by inf::TargetDistr.

Bounds

Another important task of the inf::DualVector class is to compute what we call "bounds" for short. Here is the idea. Suppose that an inf::EventTensor (the inf::DualVector owns such an inf::EventTensor through the inf::TensorWithOrbits inheritance) assigns to the event (0,0) the value 12 and to the event (0,1) the value 15. It will be useful for the branch-and-bound algorithm of the inf::TreeOpt and inf::LiveTreeOpt classes to be able to tell in advance what the highest/lowest score of an event that is only partially filled (known) would be. In our example, the partially filled event (0,?) (which can be filled to be either (0,0) or (0,1)) would have 12 as a lower bound on its score and 15 as an upper bound on its score. (By "score" we just mean the value that the inf::DualVector / inf::EventTensor assigns to it.)

The way the bounds are stored is explained in inf::DualVector::StoreBounds, which is controles whether or not an inf::DualVector computes these bounds. This is passed to inf::Constraint so that it can adequately initialize each of its inf::DualVector. Whether the bounds that are computed are upper or lower bounds is controlled with the type inf::DualVector::BoundType. The idea is that the left-hand-side inf::DualVector stored in the inf::Constraint has a positive scale factor, and since our optimization algorithms are minimization algorithms, we need the left-hand-side dual vector to compute lower bounds. The right-hand-side inf::DualVector stored in the inf::Constraint has a negative scale factor, and so it needs to compute upper bounds.

The way the bounds are computed is explained in inf::DualVector::BoundRule.

The bound mechanism is tested in the application user::dual_vector_bounds and user::opt_bounds.

An inf::DualVector is an util::Serializable, allowing to write/read them from disks, which we use for nonlocality certicates as described in inf::FeasProblem. This serialization mechanism is tested in the application user::dual_vector_io.

◆ BoundRule

typedef std::pair<QuovecIndex, std::vector<QuovecIndex> > inf::DualVector::BoundRule
private

This type describes how to compute upper/lower bounds: bound_rule.first is the quovec index to set as the min/max of the quovec elements indexed by bound_rule.second

In principle, we could simply say that the event (0,?) computes its bounds based on the min/max of the values assigned to (0,0) and (0,1). However, we can directly compute the bounds at the level of the quovec: the quovec index corresponding to (0,?) (stored in bound_rule.first) looks at the quovec indices (basically, the orbits) of (0,0) and (0,1).

◆ StoreBounds

enum class inf::DualVector::StoreBounds
strong

This controls whether or not an inf::DualVector will store some bounds.

The type of bounds is controlled with inf::DualVector::BoundType. The way the bounds are stored is as follows: the bound for the partially filled event (0,?) is stored as if the number of outcomes was one more than the "physical" number of outcomes, i.e., instead of \(\nouts\) outcome, we now have \(\nouts+1\) outcome. The last outcome represents the "unknown" outcome, denoted "?" in the documentation.

Enumerator
yes 

Do store bounds, implies a slightly increased memory usage.

no 

Do not store bounds.

◆ get_event_to_quovec_index()

std::vector< inf::QuovecIndex > const & inf::DualVector::get_event_to_quovec_index ( ) const

To map the hash of each marginal event \(\infevent \in \events\infmarg\) to the quovec index of its orbit.

This returns m_event_to_quovec_index. Recall that inf::EventTensor assigns a hash to each event it is valued on (see inf::EventTensor::get_event_hash()). Given an event \(\infevent\in\events\infmarg\) with hash i, m_event_to_quovec_index[i] identifies the orbit (or its representative) \(\act{\marggroup}{\events\infmarg}(\infevent)\). The representatives are then ordered and assigned an index m_event_to_quovec_index[i].

◆ set_from_quovec()

void inf::DualVector::set_from_quovec ( inf::Quovec const &  quovec,
inf::QuovecIndex  start_pos 
)

This sets the coefficients (image) of the inf::DualVector, including bounds if necessary.

The idea is that the underlying m_event_tensor (an inf::EventTensor) will have its coefficients set as m_event_tensor[j] = quovec[start_pos + i], where i = get_event_to_quovec_index()[j], which represents the embedding \(\altquovecembed\infmarg(\quovec)(\infevent) = \quovec(o_{\infevent})\), where \(\infevent \in \events\infmarg\) is a marginal event (assigned the hash j), \(\quovec\in\quovecspace\infmarg\) is the parameter quovec (offset with start_pos), \(\altquovecembed\infmarg(\quovec) \in \vecset\infmarg\) represents the underlying m_event_tensor, and \(o_{\infevent} = \act{\marggroup}{\events\infmarg}(\infevent)\) denotes the orbit (assigned the index i) of the marginal event \(\infevent\) .

Furthermore, this methods compute the lower/upper bound, depending on how the inf::DualVector was initialized (see inf::DualVector::StoreBounds and inf::DualVector::BoundType).

Parameters
start_posIndicates where to start reading the quovec. This is there to allow us to represent a set of quovec \(\{\quovec_\constraintname\}_{\constraintname\in\constraintlist}\) as a flat list of coefficients when we deal with a set of constraints \(\constraintlist \subset \infconstraints\).
