inf::Constraint Class Reference

#include "inf/constraints/constraint.h"

Class description

Describes a single constraint of the form \(\infq_{\infmarg_{0}\cdots\infmarg_{k-1}\infmargg} = \targetp_{\infmarg_{0}} \cdots \targetp_{\infmarg_{k-1}}\cdot \infq_{\infmargg}\).

Recall that an inflation problem is essentially of the following form: given a target distribution \(\targetp \in \targetps\) (whose nonlocality we want to test), find an inflation distribution \(\infq\in\infqs\) over the inflation graph satisfying some symmetry requirements and some constraints. The constraints are specified as a set \(\constraintlist \subset \infconstraints\) as described in inf::ConstraintSet. Each constraint is of the form \(\constraintname = (\infmarg_{0},\dots,\infmarg_{k-1},\infmargg) \in \constraintlist\), where \(\infmarg_i,\infmargg \subset \infparties\) satisfy some properties described and checked in inf::ConstraintParser. The idea is that the properties satisfied by these sets of inflation parties imply that imposing the constraint \(\infq_{\infmarg_{0}\cdots\infmarg_{k-1}\infmargg} = \targetp_{\infmarg_{0}} \cdots \targetp_{\infmarg_{k-1}}\cdot \infq_{\infmargg}\) yields a valid relaxation of the local set of the relevant inf::Network.

As described in the paper, the constraint \(\infq_{\infmarg_{0}\cdots\infmarg_{k-1}\infmargg} = \targetp_{\infmarg_{0}} \cdots \targetp_{\infmarg_{k-1}}\cdot \infq_{\infmargg}\) ends up being represented as \(\totconstraintmapelem(\infq) = 0\), where \(\totconstraintmapelem : \vecset{\infparties} \to \quovecspace{\infmarg_{0}\cdots\infmarg_{k-1}\infmargg}\). To check nonlocality certificates and enable the Frank-Wolfe algorithm that finds such nonlocality certificates, we need to be able to evaluate inner products of the form \(\inner{\quovec}{\totconstraintmapelem(\detdistr\infevent)}\), where \(\quovec \in \quovecspace{\infmarg_{0}\cdots\infmarg_{k-1}\infmargg}\) is a dual vector (which is reduced/quotiented with respect to the relevant symmetries, we call this an inf::Quovec in the code), and where \(\detdistr\infevent \in \infqs\) is the deterministic inflation distribution with support on the inflation event \(\infevent \in \infevents\).

The class inf::Constraint provides the means necessary to evaluate such inner products. Internally, it requires two inf::Marginal, one (m_lhs_inflation_marginal) to represent the left-hand-side inflation marginal \(\infq_{\infmarg_0\cdots\infmarg_{k-1}\infmargg}\) and one (m_rhs_inflation_marginal) to represent the right-hand-side inflation marginal \(\infq_{\infmargg}\). Furthermore, it stores the current dual vector \(\quovec\) as the inf::DualVector m_lhs_dual_vector. It also stores an inf::EventTensor, m_rhs_target_tensor, representing \(\targetp_{\infmarg_0}\cdots\targetp_{\infmarg_{k-1}}\). As described in the paper, we can more efficiently evaluate the right-hand-side term of the inner product using the dual vector \(\tilde\quovec\in\quovecspace{\infmargg}\), stored as m_rhs_reduced_dual_vector, which represents the contraction of \(\targetp_{\infmarg_0}\cdots\targetp_{\infmarg_{k-1}}\) with \(\altquovecembed{\infmarg_0\cdots\infmarg_{k-1}\infmargg}(\quovec)\).
