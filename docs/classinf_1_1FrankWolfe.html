Class description

The interface to the different Frank-Wolfe algorithms that we implement.

The general Frank-Wolfe algorithm as used in inf::FeasProblem works as follows:

  1. Store some inflation event \(\infevent\in\infevents\) and the associated quovec \(\quovec = \totconstraintmap(\detdistr\infevent) \in \totquovecspace\) (see inf::FeasProblem::init_frank_wolfe()). Let \(\mathcal S \leftarrow \{\quovec\}\) (although, see inf::FeasProblem::RetainEvents).
  2. Using inf::Optimizer::optimize(), minimize the inner product \(\inner{\quovec}{\totconstraintmap(\detdistr{\infevent'})}_{\constraintlist}\) for all \(\infevent' \in \infevents\).
  3. If the result of the minimization is strictly positive, then \(\quovec\) is a nonlocality certificate, as shown in the paper. Else, let \(\mathcal S \leftarrow \mathcal S \cup \{\totconstraintmap(\detdistr{\infevent^*})\}\) (see inf::FrankWolfe::memorize_event_and_quovec()), where \(\infevent^*\) is the inflation event with the minimum score.
  4. Based on \(\mathcal S\), find a new dual vector \(\quovec \in \totquovecspace\) (see inf::FrankWolfe::solve()), and go back to step 2. If no new dual vector \(\quovec\) could be found, exit the loop, concluding that the inf::FeasProblem is inconclusive.

We propose two inf::FrankWolfe algorithms to solve step 4, see inf::FrankWolfe::Algo.

  • inf::FullyCorrectiveFW is the most efficient algorithm for the typical inflation problems that we consider, which, in the landscape of general polytope membership problems, is characterized as by few dimensions (few scalar constraints), in the order of thousands, but many extremal vertices (many inflation events), in the order of billions. It requires Mosek, see https://docs.mosek.com/latest/cxxfusion/install-interface.html for details on how to install it.
  • inf::PairwiseFW is in principle more scalable for general polytope membership problems with many dimensions (say, millions of dimensions), and does not require to setup Mosek. We leave this algorithm for completeness but we expect this algorithm to be much slower than inf::FullyCorrectiveFW in applications.

To conveniently instantiate a subclass an inf::FullyCorrectiveFW or an inf::PairwiseFW.

Parameters
algoThe algorithm choice
dimensionThe dimension of the space in which the Frank-Wolfe algorithm takes place, which in our case is the dimension of dual vectors \(\quovec \in \totquovecspace\), i.e.,

\[ \sum_{\constraintname=(\infmarg_0,\dots,\infmarg_{k-1},\infmargg) \in \constraintlist} |\events{\infmarg_0\cdots\infmarg_{k-1}\infmargg} \setminus \marggroup|, \]

where \(\constraintlist \subset \infconstraints\) is the set of constraints describing the inflation problem at hand, see inf::ConstraintSet.
n_threadsThe number of threads to use. Currently, this is only used if algo == inf::FrankWolfe::Algo::fully_corrective, see inf::FullyCorrectiveFW.

This method is called with the output of the inf::Optimizer, notifying the Frank-Wolfe algorithm that a new interesting inflation event \(\infevent\in\infevents\) with associated quovec \(\quovec = \totconstraintmap(\detdistr\infevent)\) has been found.

This rescales quovec by dividing each of its component by denom and calls inf::FrankWolfe::memorize_event_and_quovec_double()

Parameters
eventThe inflation event \(\infevent\in\infevents\) associated to quovec
quovecUp to a constant scale factor, this is the dual vector \(\quovec = \totconstraintmap(\detdistr\infevent)\), where \(\infevent\) corresponds to event
denomThe scale factor that will divide each component of quovec to obtain a floating-point dual vector
