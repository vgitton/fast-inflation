inf::EventTensor Class Reference
Events » Event storage

#include "inf/events/event_tensor.h"

Class description

Stores a function from a set of inf::Event of fixed length to Num.

Conceptually, this plays the role of a function \(v \in \vecset{\set A}\) for some set \(\set A\), i.e., a function \(v : \events{\set A} \to \R\), except that we restrict the images of \(v\) to rational numbers. The idea of the inf::EventTensor is that it stores rational numbers expressed with the same denominator. Thus, it is mostly a function from inf::Event to Num, but additionally provides a denominator (see inf::EventTensor::get_denom()) and some rational arithmetic capabilities such as inf::EventTensor::simplify().

Example use:

inf::EventTensor T(3,4);
T.get_num({0,0,0}) = 1;
T.get_num({0,0,1}) = 2;
// Now, T contains
// {0,0,0} -> 1
// {0,0,1} -> 2
// {0,0,2} -> 0
// {0,0,3} -> 0
// ...
// {3,3,3} -> 0

This class:

  • is used over std::map because the latter was not so efficient (accessing an item takes longer).
  • is used internally by inf::TensorWithOrbits (and so by inf::TargetDistr and inf::DualVector).
  • is used by the end-user to specify the components of an inf::TargetDistr.
  • is tested in the user::event_tensor application.

◆ set_to_tensor_product()

void inf::EventTensor::set_to_tensor_product ( std::vector< inf::EventTensor const * > const &  factors)

Sets the current tensor to the tensor product of the elements of factors, and calls inf::EventTensor::simplify()

Before calling this method, make sure that this has the right size: there will be a soft assertion ASSERT_EQUAL(this->get_n_parties(), sum_factors_n_parties), where sum_factors_n_parties is the sum over factor in factors of factor.get_n_parties(). Furthermore, this and all factor in factors should have the same base.

Note
This method is not meant to be particularly fast.
