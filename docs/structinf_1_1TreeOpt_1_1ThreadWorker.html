#include "inf/optimization/tree_opt.h"

Class description

This struct encapsulates the data that each thread works with.

This allows in particular to conveniently initialize the storage of each thread is a single struct instantiation.

◆ n_leaves_effective

Index inf::TreeOpt::ThreadWorker::n_leaves_effective

The number of leaves perceived thanks to the branch-and-bound approach.

If the constraints passed to inf::TreeOpt::TreeOpt() have constraints.get_store_bounds == inf::DualVector::StoreBounds::no, then n_leaves_effective will just be the number of leaves of the subtree that the inf::TreeOpt::ThreadWorker considers. If instead constraints.get_store_bounds == inf::DualVector::StoreBounds::yes, meaning that the branch-and-bound approach is used, this is the number of effective leaves perceived, i.e., when a branch containing, say, 10 leaves is discarded from a non-terminal depth, this counts as one effective leave.

See also
inf::ConstraintSet::get_store_bounds()
