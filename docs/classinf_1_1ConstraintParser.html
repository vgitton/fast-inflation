inf::ConstraintParser Class Reference

#include "inf/constraints/constraint_parser.h"

Class description

To convert a human-readable formulation of inflation constraints into the necessary internal format.

Recall that, in the paper, an inflation constraint is written as \((\infmarg_0,\dots,\infmarg_{k-1},\infmargg) \in \infconstraints\) for some \(k \geq 1\), where \(\infmarg_i,\infmargg \subset \infparties\) (where \(\infparties\) denotes the set of inflation parties). This will then imply a constraint of the form \(q_{\infmarg_0\cdots\infmarg_{k-1}\infmargg} = p_{\infmarg_0} \dots p_{\infmarg_{k-1}} \cdot q_{\infmargg}\) in the inflation problems we consider (i.e., "find an inflation distribution satisfying some symmetry requirements as well as this marginal constraint"). The class inf::ConstraintParser takes care of two things:

  • It defines the format in which the user should enter inflation constraints, allowing to convert the user input into list of indices of inflation parties as defined in inf::Inflation. This is used in particular when initializing an inf::Constraint.
  • It takes care of checking that the data that was entered makes sense as an inflation constraint. This essentially consists in checking that each \(\infmarg_i\) is an injectable set ( \(\injsets\) in the paper), and that for each \(\infmarg,\infmarg' \in \{\infmarg_0,\dots,\infmarg_{k-1},\infmargg\}\), \(\infmarg\) is d-separated from \(\infmarg'\), i.e., that there is no common parent source between \(\infmarg\) and \(\infmarg'\).

Formatting of inflation constraints

We represent this with the inf::Constraint::Description type. For instance,

  • \((\infmarg_0 = \{\alice_{00},\bob_{00},\charlie_{00}\},\infmarg_1 = \{\alice_{11},\bob_{11},\charlie_{11}\},\infmargg = \emptyset)\) is represented as
    // Notice the last empty string, which represents the empty set!
    inf::Constraint::Description constraint_description = {"A00,B00,C00", "A11,B11,C11", ""};

◆ get_target_marg_names()

std::vector< inf::TargetDistr::MarginalName > inf::ConstraintParser::get_target_marg_names ( ) const

The list of target distribution marginals corresponding to each \(\infmarg_i\).

The idea is that each \(\infmarg_i\) is an injectable set, so if we strip away the copy indices from the party to just retain their type (Alice, Bor or Charlie), we get something that defines an inf::TargetDistr::MarginalName. This method allows to compute more easily the necessary marginals \(\targetp_{\infmarg_i}\).
