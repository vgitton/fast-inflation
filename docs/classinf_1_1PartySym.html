Class description

Party symmetry.

Consider a set of parties \(\set A\), such as \(\{0,1,2\}\) (contrarily to the paper, we will mostly label parties with indices for computational efficiency reasons). A party symmetry is a permutation \(\sigma \in \symgroup{\set A}\).

Symmetry action, acts on each party entrywise

The idea is that input should be thought of a list of parties (indices) from the set \(\set A\) on which the inf::PartySym \(\sigma \in \symgroup{\set A}\) acts. Then, \(\sigma\) is applied entrywise on the input, and the result is stored in output.

Note
input and output are allowed to be pointing at the same memory location. There is a soft assertion checking that the provided input and output vectors have the same size.

Symmetry action, essentially the action on events.

The parameter input, being a list, should be thought of as a function \(f : \set A \to T\), where \(T\) is the set of objects of type T (the template parameter). Then, the symmetry action reads \(\sigma(f) = f \circ \sigma^{-1}\). This describes for instance the action of party symmetries on events, denoted \(\act{\sigma}{\events{\set A}}\) in the paper, but also the action of constraint symmetries on marginal permutations, denoted \(\actt{\sigma}{\permutedmargs{A}}\) in the paper.

Note
input and output have to refer to different memory locations, and their size should match \(|\set A|\), i.e., the size of the sym parameter passed to the constructor inf::PartySym::PartySym(). This is soft-asserted.
