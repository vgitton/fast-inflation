inf::FeasProblem Class Reference
Inflation » Inflation problem specification

#include "inf/inf_problem/feas_pb.h"

Class description

This class defines inflation problems, i.e., problems of the form: given a target distribution \(\targetp\in\targetps\), is there an inflation distribution \(\infq\in\infqs\) satisfying appropriate symmetry and marginal constraints?

Defining an inf::FeasProblem requires passing an inf::TargetDistr as target distribution \(\targetp\in\targetps\) and inf::FeasOptions specifying the inflation size \(\infsize = (\infsize_0,\infsize_1,\infsize_2)\) (see inf::Inflation::Size) and the inflation constraints \(\constraintlist \in \infconstraints\) (see inf::ConstraintSet). Additionally, the inf::FeasOptions specify the methods to use to solve the inflation problem. The inf::FeasProblem is solved using a Frank-Wolfe algorithm, see inf::FrankWolfe.

This class allows to write and read nonlocality certificates from text files, see inf::FeasProblem::write_dual_vector_to_file() and inf::FeasProblem::read_dual_vector_from_file() for more details.

◆ Status

enum class inf::FeasProblem::Status
strong

The return status of a feasibility problem.

Enumerator
nonlocal 

The target distribution \(\targetp\) is provably nonlocal using exact arithmetic. This conclusion comes from the inf::Optimizer having checked that a nonlocality certificate is valid.

inconclusive 

Inconclusive (may or may not be local): up to numerical precision, the target distribution \(\targetp\) appears to be compatible with the inflation problem. Going to a larger inflation with more constraints may allow to prove the nonlocality of the distribution \(\targetp\). This conclusion comes from the inf::FrankWolfe algorithm being unable to provide a next dual vector that would be a candidate nonlocality certificate.

◆ RetainEvents

enum class inf::FeasProblem::RetainEvents
strong

One option of the inf::FeasProblem is whether or not to retain previously encountered event when updating the target distribution.

This has proven to be quite useful for performance when scanning the feasibility of a family of inf::TargetDistr. The idea is that when the inf::TargetDistr is changes a little bit, the inflation events generating the active set used in the inf::FrankWolfe algorithm are still relevant ones. It is thus recommended to use inf::FeasProblem::RetainEvents::yes when running sequential inf::FeasProblem with inf::TargetDistr that are not too different from each other. It may however be detrimental to use this option when running inf::FeasProblem with "completely different" inf::TargetDistr.

Enumerator
yes 

This is used to indicate that the inf::FrankWolfe algorithm should restart keeping the previously encountered events. This helps to quickly rule out some inf::DualVector as being potential separating hyperplanes. See inf::FrankWolfe::memorize_event_and_quovec().

no 

Do not keep the previously encountered events when restarting the inf::FrankWolfe algorithm.

◆ get_feasibility()

inf::FeasProblem::Status inf::FeasProblem::get_feasibility ( )

This is the heavy function that checks feasibility of a target distribution.

This checks the compatibility of the inf::TargetDistr \(\targetp\in\targetps\) that was passed to the constructor or last passed to inf::FeasProblem::update_target_distribution(), if applicable. The way the algorithm works is by successively invoking inf::FrankWolfe::time_and_solve() to find a next inf::DualVector \(\quovec = \{\quovec_\constraintname\}_{\constraintname\in\constraintlist} \in \totquovecspace\) that may be a potential separating hyperplane, and then to minimize the inner product \(\inner{\quovec}{\totconstraintmap(\detdistr\infevent)}_{\constraintlist}\) over all \(\infevent\in\infevents\) (actually, it is sufficient to only look at the symmetric inflation events obtained with inf::Inflation::get_symtree()) using inf::Optimizer::optimize(). If the algorithm finds a dual vector \(\quovec\) such that \(\inner{\quovec}{\totconstraintmap(\detdistr\infevent)}_{\constraintlist} > 0\) for all \(\infevent\), then this proves the nonlocality of the distribution \(\targetp\), as shown in the paper.

Returns
The feasibility status, see inf::FeasProblem::Status for more information

This allows to change the target distribution \(\targetp\in\targetps\) without recreating an inf::FeasProblem from nothing.

This methods throws an error if d has different symmetries compared to the initial distribution, see inf::Inflation::has_symmetries_compatible_with(). In this case, one should re-create a new inf::FeasProblem entirely. In the future, this may be automatized from within the method if it is common to have families of distributions with different symmetries.

Parameters

void inf::FeasProblem::log_info_and_init_optimizer ( )
private

This logs info about the options, pretty prints the constraints, and initializes the inf::Optimizer.

The reason for initializing the inf::Optimizer in this method is that it may be quite slow to initialize the inf::Optimizer if a call to inf::Inflation::get_symtree() is required. It is thus nice to first print the problem that is being solved, so that the user has a chance to realize that the problem being solved is not the one they wanted to solve before having to wait for the inf::Optimizer initialization.

◆ round_dual_vector()

inf::Quovec inf::FeasProblem::round_dual_vector ( std::vector< double > const &  dual_vector_double) const
private

This takes as input a floating-point representation of a quovec \(\quovec = \{\quovec_\constraintname\}_{\constraintname\in\constraintlist} \in \totquovecspace\) and returns a scaled-up integer quovec.

In more details, this method scales up the input quovec as much as possible by multiplying it with a positive constant, then floors each entry to obtain an integer quovec, and finally divide each entry by the GCD of all entries. This allows to the compute inner products using integer arithmetic. Implicitly, this uses the fact that multiplying or dividing a quovec by a positive constant does not change the result of a minimization of \(\inner{\quovec}{\totconstraintmap(\detdistr\infevent)}_{\constraintlist}\) over \(\infevent\), and nor does it change whether or not a dual vector \(\quovec\) is or isn't a nonlocality certificate (since this relies on \(\inner{\quovec}{\totconstraintmap(\detdistr\infevent)}_{\constraintlist}\) being positive for all \(\infevent\)).

Parameters
dual_vector_doubleA floating-point representation of a dual vector/quovec \(\quovec = \{\quovec_\constraintname\}_{\constraintlist}\)
Returns
A scale-and-rounded representation of dual_vector_double
